TASK-22: PDF Export for Financial Reports
============================================

Date: 2026-05-06
Status: COMPLETED

FILES CREATED:
--------------
1. app/Exports/FinancialReportPdf.php
   - Main PDF export class using barryvdh/laravel-dompdf
   - Methods: generate(), download(), stream()
   - Supports: simple, rigid, custom report types
   - Date range formatting with Carbon
   - Currency formatting with number_format (Indonesian Rupiah)

2. resources/views/exports/financial-report.blade.php
   - HTML template with inline CSS (dompdf-compatible)
   - Header with title "Laporan Keuangan" + period
   - Footer "Generated by W9 Cafe POS"
   - Page break support for rigid report

REPORT TYPE SUPPORT:
-------------------
1. Simple Report:
   - Ringkasan table: Total Income, Total Expense, Net, Receivables
   - Breakdown tables: Income by source, Expense by category

2. Rigid Report:
   - Income Statement section: Pendapatan, HPP, Laba Kotor, Beban Operasional, Beban Tak Terduga, Laba/Rugi Bersih
   - Cash Flow section: Arus Kas Masuk, Arus Kas Keluar, Saldo Awal, Arus Kas Bersih, Saldo Akhir

3. Custom Report:
   - Detail table: Date | Category | Type | Amount | Running Total
   - Summary: Total Income, Total Expense, Net

INDONESIAN CHARACTER SUPPORT:
-----------------------------
- All text in Indonesian language
- Currency format: "Rp 1.000.000" (Indonesian Rupiah)
- Date format: "dd M Y" (e.g., "06 May 2026")
- Labels: "Pemasukan", "Pengeluaran", "Laba/Rugi Bersih", etc.

USAGE EXAMPLE:
--------------
use App\Exports\FinancialReportPdf;
use App\Services\SimpleReportService;

$service = new SimpleReportService();
$data = $service->generate('2026-01-01', '2026-01-31');

return FinancialReportPdf::download($data, 'simple', '2026-01-01', '2026-01-31');

DEPENDENCIES:
-------------
- barryvdh/laravel-dompdf (already in composer.json)
- Carbon (Laravel built-in)
- Report services: SimpleReportService, RigidReportService, CustomReportService

NOTES:
------
- CSS uses inline styles for dompdf compatibility
- No complex CSS (flexbox, grid) - only tables and basic styling
- Page break inserted between Income Statement and Cash Flow in rigid report
- Color coding: green for positive, red for negative amounts
